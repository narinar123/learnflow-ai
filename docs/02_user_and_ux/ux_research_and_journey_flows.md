# UX Research & Journey Flows
**AI Learning & Member Management Platform — LearnFlow AI**
**Version:** 1.0 | **Date:** July 2026

---

## 1. UX Research Objectives

1. Understand the **mental models** of learners across age groups when they seek online education
2. Identify the **top friction points** that cause drop-off during onboarding and learning
3. Validate **navigation structures** and information architecture with target users
4. Evaluate **content discoverability** and course search behavior
5. Understand how users perceive and respond to **AI assistant** features in a learning context
6. Assess **gamification mechanics** effectiveness across age groups
7. Identify **accessibility barriers** for users with visual, cognitive, or motor limitations

---

## 2. Research Methods & Plan

### 2.1 Research Methods

| Method | Purpose | Sample Size | Duration |
|---|---|---|---|
| Contextual User Interviews | Understand goals, pain points, mental models | 15 participants (3 per persona) | 45–60 min each |
| Online Survey | Validate hypotheses at scale | 200+ respondents | 10–15 min |
| Usability Testing (Low-Fi) | Test wireframe navigation and flow logic | 8–10 participants | 30 min each |
| Usability Testing (High-Fi) | Validate final design before development | 10–12 participants | 45 min each |
| Tree Testing | Validate information architecture | 30+ participants | 15 min each |
| Card Sorting | Validate course categorization structure | 20 participants | 20 min each |
| Diary Study | Understand longitudinal learning behavior | 10 participants over 14 days | Passive |
| Heuristic Evaluation | Expert review against Nielsen's 10 heuristics | 3 UX experts | 2 hours each |
| WCAG 2.2 Accessibility Audit | Identify accessibility compliance gaps | Automated + 2 experts | 4 hours |

### 2.2 Participant Recruitment Criteria
- Age range: 13–65 years across all personas
- Device usage: at least 1 mobile device (iOS or Android)
- Prior eLearning experience: mix of none, some, heavy
- Geographic spread: Metro cities (Mumbai, Bengaluru, Delhi) + Tier 2 (Pune, Jaipur, Coimbatore)
- Language: English primary, Hindi optional for comprehension tests
- Compensation: ₹500 gift voucher per participant

## 5. Full User Journey Maps

### 5.1 New User Onboarding Journey

| Phase | User Actions | Touchpoints | Emotions | Pain Points | Opportunities |
|---|---|---|---|---|---|
| Discovery | Sees Instagram ad or friend recommendation | Social media, Google search | Curious, skeptical | "Is this legit?" | Strong social proof, trust signals on landing page |
| Landing | Visits marketing website, reads features | Marketing website | Interested, evaluating | Complex pricing confuses | Simple pricing table, free tier CTA prominent |
| Registration | Clicks "Get Started Free", fills form, OTP | Registration screen | Motivated, impatient | OTP delays feel long | <30s OTP, progress indicator, auto-focus |
| Role Selection | Chooses "Student" or "Professional" | Onboarding screen 1 | Excited | Unclear which to choose | Clear descriptions with examples per role |
| Goal Setting | Selects learning goal from visual cards | Onboarding screen 2 | Engaged | Too many options | Max 6 goals shown, "I'm not sure" option |
| Recommendations | Sees personalized course list | Onboarding screen 3 | Delighted or overwhelmed | Doesn't trust AI picks | Explain WHY courses are recommended |
| First Enrollment | Clicks "Start Course" | Course detail page | Motivated | Paywall surprise | Free courses visually distinct from paid |
| First Lesson | Starts first video lesson | Lesson player | Curious, eager | Video quality or buffering | Adaptive streaming, pre-load first 30s |
| Completion | Finishes lesson, XP awarded | Dashboard | Accomplished | Unsure what to do next | Clear "Next lesson" CTA, XP celebration |

### 5.2 Course Discovery & Enrollment Journey

| Phase | User Actions | Touchpoints | Emotions | Pain Points | Opportunities |
|---|---|---|---|---|---|
| Browse | Scrolls homepage, sees featured courses | Dashboard, Course catalog | Curious | Too many courses, no clear path | Curated sections: "For You", "Popular", "New" |
| Search | Types keyword in search bar | Search screen | Focused | Results feel irrelevant | Semantic search, filters, autocomplete |
| Filter | Filters by level, duration, rating | Filter panel | Analytical | Complex filters frustrate mobile users | Mobile-optimized bottom sheet filter |
| Course Detail | Reads description, checks reviews | Course detail page | Evaluating | Not enough social proof | Student counts, recent reviews, preview |
| Enroll/Purchase | Clicks "Enroll Free" or "Buy Now" | Payment screen | Committed or hesitant | Payment friction for paid | Razorpay native SDK for 1-click UPI |
| Confirmation | Gets confirmation screen + email | Success screen, email | Relieved, excited | Unsure what to do next | Direct CTA to start first lesson |

### 5.3 Active Learning Journey (Daily Session)

| Phase | User Actions | Touchpoints | Emotions | Pain Points | Opportunities |
|---|---|---|---|---|---|
| Open App | Opens app, sees dashboard | Dashboard | Habitual | Cold start feels impersonal | Daily greeting, streak shown prominently |
| Resume | Taps "Continue" on course card | Course card | Motivated | Forgets where they left off | Timestamp shown: "Resume from 4:32" |
| Watch Lesson | Watches video, takes notes | Lesson player | Focused | Distractions, slow buffering | PiP mode, offline mode, notes overlay |
| Complete Lesson | Lesson marked done, XP awarded | Lesson completion toast | Accomplished | No celebration feels hollow | Micro-animation, XP toast, "Next" auto-advance |
| Quiz Prompt | System suggests quiz after 3 lessons | Dashboard / lesson end | Curious | Feels like homework | Frame as "Test your knowledge" not "Quiz" |
| Session End | Closes app | Push notification later | Satisfied or guilty | No closure feeling | "You've earned 45 XP today!" session summary |

### 5.4 Quiz & Assessment Journey

| Phase | User Actions | Touchpoints | Emotions | Pain Points | Opportunities |
|---|---|---|---|---|---|
| Quiz Start | Taps "Take Quiz" button | Quiz intro screen | Anxious or confident | Timer pressure | Show question count + estimated time |
| Answer Questions | Selects answers, navigates between | Quiz screen | Focused | Accidental back navigation | Confirm-before-exit dialog |
| Timer Running | Watches countdown clock | Quiz screen | Stressed | Running out of time | Warning at 30% time left |
| Submit | Confirms submission | Quiz submit dialog | Relieved | No going back | "Review before submit" option |
| Results | Sees score, XP earned | Results screen | Pride or disappointment | Doesn't understand wrong answers | "Explain this answer" AI button |
| Retry | Taps retry after cooldown | Quiz retry screen | Determined | Long cooldown feels punitive | Show learning material to review first |

| Phase | User Actions | Touchpoints | Emotions | Pain Points | Opportunities |
|---|---|---|---|---|---|

| AI Chatbot | Asks AI chatbot | Chat interface | Cautiously optimistic | Chatbot doesn't understand | Smooth handoff to human agent |

## 6. Usability Testing Plan

### 6.1 Round 1: Low-Fidelity Testing (Wireframes)
- **Objective:** Validate navigation structure, information hierarchy, and core flow logic
- **Participants:** 8 people (2 per persona group: Student, Professional, Trainer, Institution)
- **Tasks:**
  1. "You want to learn Python. Find and enroll in a course."
  2. "You completed a course. Find and download your certificate."
  3. "You've forgotten your password. Reset it."
  4. "You want to contact support about a payment issue."
  5. "You're a trainer. Create a new course and add a lesson."
- **Success Metrics:** Task completion rate ≥ 80%, time-on-task within expected range
- **Tools:** Figma prototype, Maze.design, screen recording

### 6.2 Round 2: High-Fidelity Testing (Interactive Prototype)
- **Objective:** Validate visual clarity, microinteraction quality, accessibility
- **Participants:** 12 people (broader age range, including 55+ participants)
- **Tasks:** Full critical path: register → onboard → enroll → learn → quiz → certificate
- **Accessibility Focus:** Screen reader test with 1 visually impaired participant
- **Success Metrics:** NPS ≥ 8/10, SUS score ≥ 80, zero P0 usability issues

## 8. Accessibility UX Guidelines (WCAG 2.2)

| Guideline | Requirement | Implementation Notes |
|---|---|---|
| Color Contrast (Text) | Minimum 4.5:1 ratio for body text | Use contrast checker in Figma; dark mode: white text on #1E1E2E |
| Color Contrast (Large Text) | Minimum 3:1 ratio for 18pt+ text | Headings can use slightly lighter colors |
| Keyboard Navigation | All interactive elements reachable via keyboard | Tab order follows visual order; no keyboard traps |
| Focus Indicator | Visible 2px focus ring on all interactive elements | Custom CSS: `outline: 2px solid #6366F1; outline-offset: 2px` |
| Touch Target Size | Minimum 44×44pt (iOS) / 48×48dp (Android) | Apply to all buttons, tabs, list items |
| Screen Reader Labels | All images, icons, and inputs have descriptive labels | `aria-label` on icons; `alt` on all images |
| Error Identification | Errors described in text, not just color | "Email is required" not just red border |
| Reading Level | Plain English at Grade 8 reading level | No jargon in UI; tooltips for technical terms |
| Captions | All video content has captions | AI-generated + trainer-editable |
| Reflow | Content readable at 400% zoom without horizontal scroll | Flexible layouts only, no fixed-width containers |
| Drag Alternatives | All drag operations have keyboard/button alternative | Lesson reorder: drag or use up/down buttons |
| Authentication | No cognitive puzzles (no complex CAPTCHAs) | Use invisible reCAPTCHA v3 or email OTP |
